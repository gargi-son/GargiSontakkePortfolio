AWS Website Hosting

Part 1: Creating and hosting it on AWS
   The purpose of this project was twofold. The first part consisted creating a static personal website using hosting it on the AWS cloud using AWS Simple Storage Service.

Creating a website
   There are many ways of creating a website by leveraging different frameworks, libraries and programming languages. This website was created using HTML for structure and CSS for styling. This was done by using different in-built functions and class to effectively structure the webpage. Few of them are listed below:

    - The <div> tag: The division tag was used to make division of content in a web page like text, images, headers, etc. It groups elements together so that they be styled using CSS.
    - The < a href >attribute with anchor element: This tag was used to create a hyperlink on a webpage and link the webpage to other webpages. It provides an absolute reference or a relative reference as its “href” value.
    - The onClick event handler: It is an event attribute that executes a script when the button is clicked. It is also used to call a function when the button is clicked. Here, the onClick attribute was used to call the toggleFunction().
    - The <span> tag: We used this tag as an inline container to mark up a part of a text or a document. Span is similar to div but <div> is a block-level element and <span> is an inline elements.
    - The <id> attribute: This attribute specifies a a unique id for an HTML element, where each value of the attribute should be unique within the document.

Hosting on AWS:
   Once we created the web application, the next step was to make it accessible to everyone on a public server. One of the effective ways to do that was to host this website using Amazon Web Services. AWS offers cloud web hosting solutions at low cost with minimum effort. Another benefit of hosting the website on AWS was that it offers scalability of the application and has a pay-as-you-go pricing model.

How to host your website on AWS
    1) Amazon offers a plethora of services to the customers and one such service for hosting a static website is Amazon Simple Storage Service or Amazon S3.
    2) S3 offers cost-effective storage classes and easy to manage features that help organize data and configure access controls while catering to the customer requirement.
    3) The first step is to create an Amazon S3 bucket and configure the settings. Since we want our website to be publicly accessible, we will unselect the “Block all Public Access” option.
    4) We also enable Static Website Hosting under properties. Here, we specify the name of the main HTML page and an error page.
    5) Next step is to upload the files to the bucket to be hosted. The URL of the website is accessible under Static Website Hosting under properties.

TroubleShooting

  • 1. CORS error - We fixed this by adding @Cross-Origin annotation in our spring boot application right before the @RequestMapping annotation.
  • EC2 Instance Creation for Rancher - Couldn’t launch Rancher with Ubuntu 22.04 AMI. We used Ubuntu 20.04 AMI and this issue was resolved.
  • 3. HTTP ERROR - Resolved it by importing http header module in the ts file.
  • 4. Multiple connections error - Do not create multiple connections to DB.
  • 5. 401 authentication error - When trying to access the deployed application from rancher we got 401 authentication error. We resolved it by adding the node port number in the URL.
  • 6. Argument of type not assignable to parameter type 'never' error - We encountered this error while connecting the backend api’s in the angular ts file. We resolved this by creating a function and defining a variable User of type any and then calling the api’s within that function.
