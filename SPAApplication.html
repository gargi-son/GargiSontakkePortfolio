Dynamic Forms

   This project primarily focused on creating a Single Page Application in Angular using RESTful Web services and JPA/Hibernate. For the backend functionality, we utilized the Amazon Relational Database Service to persist and read data to/ from a relational database.

Front-end Functionality
  The single page application is developed using VSCode as a Student Survey Form in Angular. We leveraged the Node.js functionality along with Angular to develop this project. It has the following layout:
1) Welcome page, that has two links that redirect the user to a Form and a Survey list page.
2) Survey Form Page, that allows a prospective student to fill out a survey form.
3) Survey Display Page, allowing them to view all the surveys taken to date.
These are created in the form of different angular components, which then contain the code files for each functionality. To have effective routing and navigation within the application, we utilized typescript and imported routing modules likes RouterModule and Routes from Angular.

Developing the Backend —> Creating a Database using Amazon RDS + Developing RESTful Web Services
  We used Amazon’s Relational Database Service to create a MySQL database. Restful web services act as a bridge that connects the front end to the back end. For this part of the project, we created a Spring Boot application and imported it as a Maven Project. Using Eclipse as our main IDE, we created classes for controller, service and entity with all its properties. The controller acts as a starting point and has a @RequestMapping annotation that indicated the API to be hit. We also defined the methods that we used like GET, POST, etc. Another annotation that we used was the @Autowire that avoided the creation of objects. The flow of the data is as follows:



The data object layer is where we write the SQL statements within the functions. We also establish connections to the database and the database connection properties are mentioned in the application.properties file.

Containerizing/Deploying the Application
  The first step in containerizing an application is Creating a Docker Image and Pushing it to DockerHub. We do that by first creating a Docker file that has a bunch of executable commands, building that image and then pushing it to DockerHub. Next Step in the process is creating an EC2 instance on the Amazon Web Services Management Console and logging into it via the terminal to install Docker on the instance. We also create an Identity and Access Management User on the console. To deploy the containerized application on Kubernetes, we use the Rancher Tool. We first start Rancher on the EC2 machine to get the login credentials. Next, after logging in to the Rancher, we create a Kubernetes Cluster, and then deploy our container image from there. This is done by defining a bunch of parameters like Node Ports, Number of Replicas, container image names and so on.

TroubleShooting

  • CORS error - We fixed this by adding @Cross-Origin annotation in our spring boot application right before the @RequestMapping annotation.
  • EC2 Instance Creation for Rancher - Couldn’t launch Rancher with Ubuntu 22.04 AMI. We used Ubuntu 20.04 AMI and this issue was resolved.
  • HTTP ERROR - Resolved it by importing http header module in the ts file.
  • Multiple connections error - Do not create multiple connections to DB.
  • 401 authentication error - When trying to access the deployed application from rancher we got 401 authentication error. We resolved it by adding the node port number in the URL.
  • Argument of type not assignable to parameter type 'never' error - We encountered this error while connecting the backend api’s in the angular ts file. We resolved this by creating a function and defining a variable User of type any and then calling the api’s within that function.
